# JAV New Release Tracker (JRT) - 详细设计文档

## 1. 项目概述
**项目目标**：针对特定 Jav 网站，自动化抓取指定女优在特定日期后的新发布作品，生成报告并有序地在浏览器中预览。
**核心痛点**：消除手动刷新页面、重复核对日期、繁琐点击详情页的机械劳动。
**使用场景**：个人本地运行的自动化脚本。

---

## 2. 核心功能 (Core Features)
1.  **增量抓取**：基于“上次运行时间”或“手动输入时间”过滤新片。
2.  **精准定位**：通过女优固定子域名（UID）直接访问目标主页，不进行全站泛搜索。
3.  **智能过滤**：自动剔除标题中含 `【VR】` 的条目（可扩展）。
4.  **聚合报告**：生成以女优为分类单位的 Markdown 格式预览文件。
5.  **交互式唤起**：按女优聚合顺序，分批次（每批 20 个）在默认浏览器中打开详情页。
6.  **状态保存**：运行成功后自动更新“最后运行时间”，确保下次增量更新的衔接。

---

## 3. 技术栈建议 (Technical Stack)
-   **语言**：Python 3.10+
-   **自动化/爬虫**：Playwright (支持 Chromium + Stealth 插件)
-   **HTML 解析**：BeautifulSoup4
-   **终端交互**：Rich (提供美化的控制台输出与交互)
-   **时间处理**：Arrow 或 Datetime
-   **配置文件**：YAML (存放女优名单与代理设置)
-   **状态记录**：JSON (记录上次运行日期)

---

## 4. 数据结构设计 (Data Structures)

### 4.1 配置文件 (`config.yaml`)
```yaml
settings:
  proxy: "http://127.0.0.1:7890"  # 代理服务器地址
  batch_size: 20                  # 每批打开的链接数
  headless: false                 # 是否隐藏浏览器窗口

stars:
  - { name: "三上悠亚", id: "12xc" }
  - { name: "深田咏美", id: "ok12" }
```

### 4.2 状态文件 (`state.json`)
```json
{
  "last_run_date": "2023-10-25"
}
```

## 5. 核心业务流程 (Workflow)

### 第一阶段：初始化与环境检查
1.  加载 `config.yaml` 和 `state.json`。
2.  检查代理可用性。
3.  **控制台交互**：展示 `last_run_date`，询问用户：“1. 使用上次日期 2. 手动输入新日期 (YYYY-MM-DD)”。

### 第二阶段：执行抓取
1.  启动 Playwright 浏览器实例，注入 Stealth 插件。
2.  **循环女优列表**：
    -   构造 URL: `https://www.javbus.com/star/[ID]`。
    -   随机休眠 (3-7秒) 模拟真实人类行为。
    -   解析页面：
        -   提取标题、详情页 URL、发布日期。
        -   **过滤规则 A**：如果日期 < 目标日期，立即停止抓取该女优（假设页面有序）。
        -   **过滤规则 B**：如果标题包含 `【VR】`，跳过该条目。
    -   将结果按女优姓名存入字典。

### 第三阶段：结果输出与交互
1.  **生成 Markdown**：
    -   遍历字典，写入 `report.md`。
    -   如果某女优无更新，在对应标题下标注“暂无更新”。
2.  **分批打开链接**：
    -   将所有新 URL 放入队列。
    -   `while` 循环：
        -   切片取前 20 个 URL，调用 `webbrowser.open()`。
        -   控制台阻塞等待，显示：“已打开 X/Y，输入 'y' 继续下一批，其他键退出”。
3.  **收尾**：若流程正常结束，更新 `state.json` 的日期为今日。

## 6. 关键风险与应对 (Safety & Anti-bot)
-   **Cloudflare 验证**：通过 Playwright 的非无头模式（Headless=False）允许手动介入验证。
-   **特征抹除**：使用 `playwright-stealth` 绕过 `webdriver` 检测。
-   **频率限制**：
    -   女优维度：页面级请求设置随机延迟。
    -   全局：设置合理的超时时间（Timeout）。
-   **异常处理**：若单个女优主页解析失败（如 404 或超时），记录错误并跳到下一个，不中断全局逻辑。
